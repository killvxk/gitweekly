# 自动化周报生成工具 - 使用说明

## 📋 功能概述

这个工具已经从 Python 脚本转换为 **Claude Code Skill**，可以完全自动化地：
1. 从 Git 提交历史生成周报 markdown 文件
2. 使用 Claude Code 当前会话为 GitHub 项目生成中文描述（**零成本**）
3. 自动更新周报文件中的项目说明

## 🚀 快速开始

### 方式 1：使用 Claude Code Skill（推荐）

**无需 API 密钥，零成本！**

在 Claude Code 中直接说：
- "生成周报"
- "处理文档链接"
- "更新 docs.md"

Skill 会自动：
- 询问运行模式（全自动生成周报 / 处理源文件）
- 确认参数（缓存文件路径、最多处理链接数）
- 提取 URL 并生成中文描述
- 更新文件并提交到 git

### 方式 2：使用原 Python 脚本（已废弃）

**注意**：`auto_weekly.py` 已被删除，功能已完全迁移到 Claude Code Skill。

如需使用原脚本功能，请使用 Claude Code Skill。

## 🎯 两种工作模式

### 模式 1：全自动生成周报

从 git 历史提取本周变更 → 生成周报文件 → 自动提交

**触发方式**：
```
用户: 生成本周的周报
Claude: [触发 auto-weekly skill]
        📅 周期: 2026-03-03 ~ 2026-03-09
        [处理中...]
        ✓ 周报已生成：weekly/2026-03-03_2026-03-09.md
```

### 模式 2：处理源文件

读取 markdown 文件 → 提取 URL → 生成描述 → 转换为表格

**触发方式**：
```
用户: 更新 README.md 中的链接描述
Claude: [触发 auto-weekly skill]
        [处理中...]
        ✓ 处理完成：6 个 URL
```

## 🔍 工作原理

### 完整流程（模式 1）

```
Git 提交记录
    ↓
按周分组（周一-周日）
    ↓
提取新增的 GitHub 链接
    ↓
生成 markdown 周报文件
    ↓
检查缓存
    ↓
使用 WebFetch 获取 README（无 API 限制）
    ↓
Claude Code 生成中文摘要（15-25 字）
    ↓
更新 markdown 文件
    ↓
保存到缓存 + git 提交
```

### 优化特性

✅ **零成本**
- 使用 Claude Code 当前会话
- 无需 Anthropic API 订阅
- 无需 API Key

✅ **智能缓存**
- 所有描述缓存在 `links_cache/descriptions_cache.json`
- 重复运行不会重复生成
- 支持增量更新

✅ **无 GitHub API 限制**
- 使用 WebFetch 工具直接访问
- 不受 GitHub API 速率限制
- 更快、更稳定

## 📝 描述生成规则

生成的中文描述必须符合：
- **长度**：15-25 字
- **格式**：`<核心功能>的<技术栈/领域>工具/库/框架`
- **要求**：简洁明了，突出核心功能

**优秀示例**：
- ✓ "macOS系统遥测事件生成框架，用于EDR检测能力验证和安全测试"
- ✓ "支持120+协议的高性能服务指纹识别工具"
- ✓ "Windows系统AV/EDR规避技术研究与代码集合"

**不良示例**：
- ✗ "一个很棒的项目"
- ✗ "GitHub 仓库"

详细规则请参考：`.claude/skills/auto-weekly/references/description-rules.md`

## ⚙️ 配套脚本

### cache_query.py - 查询缓存

```bash
python3 .claude/skills/auto-weekly/scripts/cache_query.py "<url>"
```

**返回值**：
- 退出码 0：找到缓存，输出描述
- 退出码 1：未找到缓存
- 退出码 2：错误

### cache_write.py - 写入缓存

```bash
python3 .claude/skills/auto-weekly/scripts/cache_write.py "<url>" "<description>"
```

**返回值**：
- 退出码 0：写入成功
- 退出码 1：写入失败

详细 API 文档：`.claude/skills/auto-weekly/references/script-api.md`

## 🐛 故障排除

### 问题 1：Python 命令不存在

某些系统使用 `python` 而非 `python3`。Skill 会自动检测可用命令。

### 问题 2：缓存文件损坏

脚本会自动备份为 `.bak` 文件并重新创建。

### 问题 3：URL 抓取失败

会自动标记为 `__DELETED__` 并写入缓存，跳过该 URL。

### 问题 4：描述质量不理想

可以手动编辑 `links_cache/descriptions_cache.json`：

```json
{
  "https://github.com/example/repo": "修改后的更好描述"
}
```

然后重新运行 skill 更新到 markdown 文件。

详细故障排除：`.claude/skills/auto-weekly/references/troubleshooting.md`
